Utiliser LM Studio avec GoudAI

Guide pas-à-pas pour faire tourner des modèles IA gratuits (Llama, Mistral, Qwen…) sur ton PC et les utiliser depuis GoudAI, même depuis ton téléphone.

Pour qui ? Débutants total. Aucune connaissance technique requise — chaque commande est copiable-collable. Compte 15-30 minutes de mise en place la première fois.

Comprendre ce qu'on fait

L'objectif : faire tourner un modèle IA directement sur ton PC (gratuit, privé) et y accéder depuis GoudAI peu importe où tu es (ton PC, ton téléphone, un café…).

Trois briques se complètent :

BriqueRôle
LM StudioApplication qui télécharge et fait tourner les modèles IA sur ton PC.
Cloudflare TunnelCrée une URL publique HTTPS qui pointe vers ton LM Studio. Permet d'y accéder depuis Internet sans toucher à ta box.
GoudAIL'interface chat qui envoie tes messages au modèle via le tunnel.
Avantages : 100% gratuit (pas de coût par token), privé (rien ne sort de ton PC sauf vers Cloudflare qui ne lit pas le contenu), accessible mobile.

Pré-requis matériel

  • Un PC Windows / macOS / Linux
  • Idéalement un GPU NVIDIA avec 8 Go de VRAM ou plus (sinon ça tourne sur CPU mais c'est lent)
  • 10 à 20 Go d'espace disque par modèle téléchargé
  • Une connexion internet stable

Pas de GPU ? Tu peux quand même tester avec un petit modèle (3B paramètres) qui tournera sur CPU à 5-15 tokens/sec.

1Installer LM Studio

  1. Va sur https://lmstudio.ai
  2. Clique Download — la bonne version pour ton OS est détectée automatiquement
  3. Lance l'installeur, accepte les paramètres par défaut
  4. Ouvre LM Studio quand l'install est terminée

2Télécharger un modèle

  1. Clique sur l'icône 🔍 Discover dans la sidebar gauche
  2. Choisis un modèle selon ton PC :
Modèle recommandéTailleAdapté à
Llama 3.2 3B Instruct~2 GoPC modeste / CPU seul
Qwen 2.5 7B Instruct~5 GoPC moyen avec GPU 8 Go
Mistral Small 24B / Devstral~14 GoPC puissant avec GPU 16 Go+
Gemma 4 e2b/e4b2-4 GoPour tester rapidement
  1. Sélectionne la variante Q4_K_M (bon compromis qualité / taille) sauf si tu sais ce que tu fais
  2. Clique Download — attends la fin du téléchargement (peut prendre plusieurs minutes selon ta connexion)
Astuce : le suffixe Q4_K_M est une "quantification" — le modèle est compressé pour tenir en mémoire. Q4 = 4 bits par poids, K_M = méthode de compression optimisée. Suffisant pour 99% des usages.

3Charger le modèle

  1. Clique sur l'icône 📁 My Models dans la sidebar
  2. Trouve ton modèle téléchargé dans la liste
  3. Clique Load à côté du nom du modèle
  4. Attends quelques secondes — la barre indique "Loaded" quand c'est prêt
Si erreur "Not enough VRAM" : ton GPU n'a pas assez de mémoire. Essaie un modèle plus petit, ou ajuste le slider "GPU Offload" pour partager entre GPU et RAM.

4Démarrer le serveur LM Studio

  1. Ouvre une fenêtre PowerShell (Windows) ou Terminal (macOS/Linux)
  2. Tape exactement :
lms server start --cors --port 2903

Tu dois voir s'afficher :

W CORS is enabled. This means any website you visit can use the LM Studio server.
Success! Server is now running on port 2903
Le serveur tourne ! Laisse cette fenêtre PowerShell ouverte. Si tu la fermes, le serveur s'arrête.

Si lms n'est pas reconnu, active la CLI une fois :

& "$env:USERPROFILE\.cache\lm-studio\bin\lms.exe" bootstrap

Puis ferme PowerShell, rouvre-la, et réessaie la commande lms server start.

5Installer Cloudflare Tunnel

Cloudflare Tunnel crée une URL HTTPS publique qui pointe vers ton LM Studio. Gratuit, pas de compte requis pour l'usage rapide.

Dans PowerShell :

winget install cloudflare.cloudflared

Attends la fin (51 Mo à télécharger). Puis ferme PowerShell et rouvre-la pour rafraîchir le PATH.

macOS : brew install cloudflared
Linux : voir documentation Cloudflare.

6Lancer le tunnel

Dans une NOUVELLE fenêtre PowerShell (en garder 2 ouvertes : une pour lms, une pour cloudflared) :

cloudflared tunnel --url http://localhost:2903

Après quelques secondes tu obtiens :

+--------------------------------------------------------------------------------------------+
|  Your quick Tunnel has been created! Visit it at:                                          |
|  https://acoustic-nvidia-compressed-alloy.trycloudflare.com                                |
+--------------------------------------------------------------------------------------------+

👉 Copie cette URL — c'est elle que tu vas mettre dans GoudAI. Garde la fenêtre PowerShell ouverte (sinon le tunnel meurt).

L'URL change à chaque relance de cloudflared. Pour avoir une URL fixe permanente, il faut un compte Cloudflare gratuit + tunnel nommé (voir section Persistance).

7Configurer GoudAI

Cette étape se fait depuis n'importe quel appareil où tu utilises GoudAI : ton PC, ton téléphone, un autre PC…

  1. Ouvre GoudAI
  2. Clique ⚙️ Configuration dans la sidebar
  3. Onglet Clés API
  4. Descends jusqu'à la section SERVEUR LOCAL (LM STUDIO / OLLAMA)
  5. Dans le champ URL, colle ton URL Cloudflare (commence par https://)
  6. Laisse Token optionnel vide pour l'instant
  7. Clique Tester et découvrir les modèles

Tu dois voir apparaître à côté du bouton :

✓ N modèles disponibles (en vert)

Si oui, clique Enregistrer tout en bas de la modale.

8Utiliser ton modèle

  1. Dans le sélecteur de modèles texte (en haut de la zone de chat), choisis ton modèle local — il apparaît avec le badge Local
  2. Écris ton message comme d'habitude
  3. La réponse est générée par ton PC, gratuitement, en local
Coût affiché : Gratuit — aucun token facturé puisque tout tourne chez toi.

Sécurité (recommandé)

L'URL Cloudflare est devinable en théorie (mots aléatoires). Pour empêcher quelqu'un qui tomberait dessus d'utiliser ton serveur :

Activer l'authentification dans LM Studio

  1. LM Studio → onglet Developer
  2. Stop le serveur si nécessaire
  3. Settings serveur → cocher Require API Key
  4. Une clé du type sk-lm-xxxxxxxx s'affiche — copie-la
  5. Redémarre : lms server start --cors --port 2903

Reporter la clé dans GoudAI

  1. Config → Clés API → Serveur local
  2. Champ Token optionnel → colle ta clé LM Studio
  3. Clique Tester → doit toujours afficher "✓ N modèles disponibles"
  4. Enregistrer
Ne partage cette clé avec personne, ni par mail, ni par chat. Elle donne accès complet à ton serveur LM Studio.

Dépannage

"Failed to fetch" / "0 modèles"

Cause la plus fréquente : CORS désactivé. Vérifie que tu as bien lancé le serveur avec --cors :

lms server stop
lms server start --cors --port 2903

"API token required"

Tu as activé Require API Key dans LM Studio mais oublié de copier la clé dans le champ Token GoudAI. Récupère la clé dans LM Studio et colle-la.

Le tunnel ne marche plus

Vérifie que la fenêtre PowerShell cloudflared est toujours ouverte. Si elle s'est fermée ou si tu as redémarré le PC, l'URL est morte → relance le tunnel et utilise la nouvelle URL dans GoudAI.

Le modèle ne répond pas (timeout)

Premier message après chargement = "cold start" du GPU, peut prendre 10-20s. Les messages suivants sont rapides. Si vraiment ça ne répond jamais, regarde l'onglet Developer de LM Studio pour voir les logs.

Sur Android, ça ne marche pas

Vérifie que tu utilises bien l'URL Cloudflare HTTPS, pas une IP locale (localhost ou 192.168.x.x ou 100.x.x.x Tailscale). Seule une URL publique HTTPS marche depuis un appareil distant.

Persistance après redémarrage

Par défaut, après chaque redémarrage du PC il faut :

  1. Rouvrir LM Studio
  2. Recharger le modèle
  3. Relancer lms server start --cors --port 2903
  4. Relancer cloudflared tunnel --url http://localhost:2903
  5. Mettre à jour la nouvelle URL dans GoudAI

Pour éviter ça, deux options avancées :

Option A — Démarrage automatique LM Studio

LM Studio → Settings (icône ⚙ globale) → activer "Start with Windows". Configure aussi Just-in-Time Loading pour que le modèle se charge à la première requête.

Option B — Tunnel Cloudflare permanent (URL fixe)

Crée un compte Cloudflare gratuit puis dans PowerShell :

cloudflared tunnel login
cloudflared tunnel create lmstudio
cloudflared tunnel route dns lmstudio lmstudio.tondomaine.com

Crée un fichier %USERPROFILE%\.cloudflared\config.yml :

tunnel: lmstudio
credentials-file: C:\Users\TON-NOM\.cloudflared\TUNNEL-ID.json
ingress:
  - hostname: lmstudio.tondomaine.com
    service: http://localhost:2903
  - service: http_status:404

Puis installe le tunnel comme service Windows (démarre tout seul au boot) :

cloudflared service install

Désormais tu as une URL fixe et permanente : https://lmstudio.tondomaine.com. Tu la mets dans GoudAI une fois pour toutes.

Pas de domaine ? Tu peux acheter un .com pour ~10€/an chez Cloudflare Registrar (à prix coûtant). Ou utiliser un sous-domaine gratuit type DuckDNS routé vers ton tunnel.

Récapitulatif après mise en place

Chaque fois que tu veux utiliser LM Studio + GoudAI :

  1. LM Studio est lancé, modèle chargé (auto si tu as activé l'option A)
  2. Serveur LM Studio actif (lms server start --cors --port 2903 dans une PowerShell)
  3. Tunnel Cloudflare actif (cloudflared tunnel --url http://localhost:2903 dans une 2e PowerShell)
  4. L'URL du tunnel est saisie dans GoudAI Config Clés API
  5. Tu sélectionnes ton modèle local dans GoudAI et tu chattes 🚀
